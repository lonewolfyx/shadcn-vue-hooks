## Getting Started

### Contribution Steps

Follow these steps to contribute to the project:

#### 1. Fork the Repository

Click the "Fork" button at the top right of the [GitHub repository](https://github.com/lonewolfyx/shadcn-vue-hooks) to create your own copy.

#### 2. Clone Your Fork

Download your forked repository to your local machine:

```bash
git clone git@github.com:lonewolfyx/shadcn-vue-hooks.git
cd shadcn-vue-hooks
```

#### 3. Install Dependencies

We use `pnpm` as our package manager. Install all project dependencies:

```bash
# Install pnpm if you don't have it
npm install -g pnpm

# Install project dependencies
pnpm install
```

#### 4. Start Development Server

Run the development server to test your changes:

```bash
# Start the development server
pnpm dev

# Or use the specific script
pnpm run dev
```

The development server will start at `http://localhost:3000` by default.

## Development Workflow

### Creating a New Hook

1. **Implement the Hook**:
   ```bash
   # Create implementation files
   touch app/registry/composable/{yourHookName.ts}
   ```

2. **Write Documentation**:
   ```bash
   # Create documentation file
   touch content/docs/2.composables/yourHookName.md
   ```

3. **Update Documentation**:
   - Add your hook to the [getting started guide](./1.introduction.md)
   - Update any relevant documentation

### Making Changes

1. **Create a New Branch**:
   ```bash
   git checkout -b feature/your-hook-name
   ```

2. **Make Your Changes**:
   - Implement your hook or fix
   - Update documentation
   - Test thoroughly

3. **Commit Your Changes**:
   ```bash
   git add .
   git commit -m "feat: add new hook - YourHookName"
   ```

4. **Push to Your Fork**:
   ```bash
   git push origin feature/your-hook-name
   ```

5. **Create a Pull Request**:
   - Go to your fork on GitHub
   - Click "New Pull Request"
   - Provide a clear description of your changes

## Coding Standards

### Code Style

- **TypeScript**: All code must be typed
- **Vue 3 Composition API**: Use composables and script setup syntax
- **ESLint**: Follow the project's ESLint configuration
- **Prettier**: Use Prettier for code formatting

### Hook Requirements

- **Type Safety**: Full TypeScript support with proper typing
- **Documentation**: Comprehensive JSDoc comments
- **Examples**: Clear usage examples in documentation
- **Testing**: Include test cases where applicable

### Documentation Standards

- **Markdown Format**: Use proper Markdown syntax
- **Examples**: Include working code examples
- **API Documentation**: Complete API reference
- **Installation Instructions**: Clear setup steps

## Submitting Contributions

### Pull Request Guidelines

- **Descriptive Title**: Clearly state what your PR does
- **Detailed Description**: Explain why and what you changed
- **Screenshots**: Include screenshots for UI changes
- **Tests**: Ensure all tests pass
- **Documentation**: Update relevant documentation

### What We're Looking For

- **New Hooks**: Reusable composables for common patterns
- **Bug Fixes**: Issues with existing hooks or documentation
- **Documentation**: Improved examples, API docs, or guides
- **Performance**: Optimizations and improvements
- **Type Safety**: Better TypeScript integration

## Getting Help

If you need help contributing:

- **GitHub Issues**: Report bugs or request features
- **GitHub Discussions**: Ask questions or discuss ideas
- **Documentation**: Check existing guides and examples
